如果勉强算一件事的话那就是写作吧。我小学的语文老师总是会将我的作文当作范文，但是她不知道的是我的作文大部分的情节都来自一本《中学生满分作文大全》。

过了一个小时面试官回来了，嗯面善的秃头老登，一看就是老程序员了。开始进入拷打流程了，你这个代码包括题目给出的示例它的问题在于这些坐标都是正数的，如果是负数呢？负数的话用min，max还能判断吗？

给我问懵了，啥玩意啊，考虑负数的话是不是要考虑这两个矩形在不在一个象限，但是老登说不对，如果是了解CV的同学会有更好的解法。我想半天没想出来这个和CV的算法有什么关系，他们题目给的图也是逆天，图给我画好了喂。我看半天一点思路没有，只好求放过。  
后面我回来复盘发现千问的代码没问题的，就算是有负数的话，如果这两个矩形不相交它算出来的重叠区域的width/height也会是负数的。这在后面的 `if width <= 0 or height <= 0:` 中会被检测出来。

```Python
def normalize(rect):
    x1, y1, x2, y2 = rect
    return [
        min(x1, x2),
        min(y1, y2),
        max(x1, x2),
        max(y1, y2)
    ]

def overlap_area(rect1, rect2):
    x1, y1, x2, y2 = normalize(rect1)
    x3, y3, x4, y4 = normalize(rect2)

    left = max(x1, x3)
    top = max(y1, y3)
    right = min(x2, x4)
    bottom = min(y2, y4)

    width = right - left
    height = bottom - top

    if width <= 0 or height <= 0:
        return -1

    return width * height
```
进入第二题，这里确实是我想不出来了。他出了一个变式题，就是说如果输入字典中有重复的字符，希望输出字典中对应的那个字符应该是包含输入数组中的匹配的字符的所有输入字段。比如输入是['a','b','a'],查找['a','b'],输出[[0,2],1]。  
因为我是用index.()直接拿的下标，我没想过如果有重复的字符怎么把所有的下标都拿出来，我第一时间想到的只有for循环（菜鸡是这样的，遇事不决就迭代），然后老登说不行这样就o(n^2)了。第二题，落败！
